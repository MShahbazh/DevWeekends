In React, **props** (properties) are read only arguments passed into components to transfer data from a parent component to the child component.
# Rules
- **Unidirectional Flow**: Data moves in one direction only, flowing downward from parent to child.
  
- **Immutable (Read-Only)**: A child component cannot modify the props it receives. If you need to change data over time, you must use **State** instead.
  
- **Any Data Type**: You can pass strings, numbers, booleans, arrays, objects, and even functions through props
# Passing Rules
- **Strings**: Use regular **quotes** `""`.
- **Numbers**: Wrap them in **curly braces** `{}`.
- **Booleans**: Wrap them in **curly braces** `{}`, or write the name alone for `true`.
- **Objects**: Wrap them in **double curly braces** `{{}}`
- **Arrays**: Wrap them in **curly braces** `{[]}`
# Example
Take an Example of `App()` Component
```javascript
function App(props) {
  return (
    <>
      <h1 className='flex flex-col align-center justify-center p-5 rounded-2xl bg-green-600 '>Tailwind Test {props.obj.world}</h1>
    </>
  )
}
```
Or Object Destructuring method:
```javascript
function App({name,obj,bool}) {
  return (
    <>
      <h1 className='flex flex-col align-center justify-center p-5 rounded-2xl bg-green-600 '>Tailwind Test {obj.world}</h1>
    </>
  )
}
```
The component takes one Object which then contains all the given variables, so we pass it like
```javascript
createRoot(document.getElementById('root')).render(
  <StrictMode>
    <App name="Hello" obj={{world:"World"}} bool={true}/>
  </StrictMode>,
)
```
So `props` structure would look like this
```javascript
props = {
	name:"Hello",
	obj:{
		world:"World"
	},
	bool:true
}
```

# Note: 
The name `props` is not mandatory, we can use whatever name we want to. The only thing to remember is that first argument of React Component is a standard JS function parameter which is an Object by nature.
