**React is a free open source JavaScript library used for building User Interfaces (UIs), primarily for single page application.**

React does not include a router by default. We must use third party libraries like React Router for page navigation. At the end of the day React breaks into HTML, CSS and JS and inject these files. 

- **Component Based Architecture:**
  We  build reusable, isolated pieces of UI components
  
- **Virtual DOM:**
  React updates a lightweight copy of DOM first. This makes page rendering highly efficient and fast
  
- **Declarative UI:**
  We describe how the UI should look for a specific state. React automatically updates view when data is changed
  
- **JSX Syntax:**
  It uses HTML like syntax which allows us to code directly inside the JS file
  
- **Unidirectional Data Flow**:
  Data moves down from parent to child components. This makes debugging and tracking errors predictable.

# Setting Up: 

## Method 1: Using npx (For Production Level)
`npm` is the node package manager for node. `npx` is node package executor for just executing without installing the package. To initialize through `npx` we can run the command. 
```text
npx create-react-app [name]
```
to start
```text
npm run start
```
However, it takes significant amount of time. 

### Features
#### Script Injection: 
The script `.js` file is injected in `public/index.html` file dynamically. So there are no script tags written inside it. `npx` uses **Webpack bundle** to handle all this. At runtime, Webpack bundles your code and dynamically injects a `<script src="/static/js/bundle.js">` tag into the HTML.

#### File Extension: 
This method allows us to write HTML like JSX syntax directly inside the standard `.js` files. But it allows `.jsx`. 


## Method 2: Using Vite (For Client Side)
Command: 
```text
npm create vite@latest
```
After that, we can start it using 
```text
npm run dev
```
But this method does not give us `node_modules`, to get this we need to run 
```text
npm i
```
This method uses `.jsx` files. 

### Features
#### Script Injection: 
The `index.html` file is in the root directory and already has the `<script type="module" src="/src/main.jsx">` tag manually hardcoded

#### File Extension: 
This method enforces a strict rule: if a file contains a JSX markup, it must use the `.jsx` extension. Plain `.js` file are strictly reserved for pure JS logic.
