React Router is a an external library used to implement client side navigation and URL management in React Applications. React does not include built in routing, so React Router is there to swap UI components dynamically based on browser's current path which eliminates the need for full page reloads. 

# Installation 
It is added into project using the command
``` bash
npm i react-router-dom
```

# Functions
## Link
`Link` is used for general navigation where we want to take user in another page. It is an equivalent of `a` tag. The only difference is that `a` tag reloads the whole page while `Link` doesn't. 
### Syntax
```javascript
import {Link} from 'react-router-dom';
<Link to="/about">About Us</Link>
```

## NavLink
`NavLink` is just like `Link` but it provides a better functionality and is used for headers, footers, sidebars or navigation bars. It acts just like `a` tag but with one advantage : it knows when the current browser matches its Link destination. When the match happens it can change its styling to highlight itself so that the user knows it is on that page. 
Instead of passing a simple string to `clasName`, we pass an arrow function. React Router gives it a `isActive`variable which is used with ternary `?:` to swap styles. 
### Syntax
```javascript
import { NavLink } from 'react-router-dom';
<NavLink to="/about" className={({ isActive }) => isActive ? "text-orange-500 font-bold" : "text-gray-500" } > About </NavLink>
```
## Outlet
The **`<Outlet />`** component is a placeholder that tells React Router exactly **where to render child routes** inside a parent layout component
```javascript
import { Header, Footer } from '../components/index'
import { Outlet } from 'react-router-dom'

const Layout = () => {
  return (
    <>
      <Header />   {/* Always stays at the top */}
      
      <main>
        <Outlet /> {/* Dynamic pages render right here! */}
      </main>
      
      <Footer />   {/* Always stays at the bottom */}
    </>
  )
}
```

## Router Configuration 
Router is created using the `createBrowserRouter`. It contains an array of route configuration objects. The parent holds the layout and children array defines what pages swap inside the `<Outlet/>`
```JSX
import { createBrowserRouter } from 'react-router-dom';
import Layout from './Layout'; // The file containing Header, Outlet, Footer
import App from './App';       // Your Home Page content
import About from './About';   // Example About Page
import Contact from './Contact'; // Example Contact Page

// Defining and Creating the router engine
const myRouter = createBrowserRouter([
  {
    path: '/',              
    element: <Layout />,     
    children: [             
      {
        path: '',          
        element: <App />
      },
      {
        path: 'about',     
        element: <About />
      },
      {
        path: 'contact',   
        element: <Contact />
      }
    ]
  }
]);
```
Another easier way of creating it :
```JSX
const router=createBrowserRouter(
  createRoutesFromElements(
    <Route path='/' element={<Layout/>}>
      <Route path='' element={<Home/>}/>
      <Route path='about' element={<About/>}/>
      <Route path='contact' element={<Contact/>}/>
      <Route path='user' element={<User/>}/>
    </Route>
  )
)
```

## Router Provider
`<RouterProvider/>` is a component that officially activates and runs the route engine in the app. 
```JSX
const router = createBrowserRouter([])
<RouterProvider router={router} />
```
## useParams()
`useParams()` is a custom React hook provided by React router that allows components to read dynamic values directly from the current URL address bar. 
### Use
First define the variable in route path. it is defined by adding a column. 
```JSX
<Route path="user/:id" element={<User />} />
```
Now read the variable inside the component using this hook 
```JSX
import { useParams } from 'react-router-dom';

function User() {
  const { id } = useParams(); 
  return (
    <div>
      <h1>User Profile Page</h1>
      <p>Now viewing user with Database ID: {id}</p>
    </div>
  );
}
```

## Loader
A loader is a feature in React Router that fetches data for a page before page even renders on the screen. 
### Usage 
First define the function 
```JSX
async function userLoader({ params }) {
  const res = await fetch(`https://example.com{params.id}`);
  return res.json(); 
}
```
Attach it to the Route
```JSX
<Route 
  path="user/:id" 
  element={<User />} 
  loader={userLoader} 
/>
```
Now in order to grab the fetched data we can use the `useLoaderData()`
```JSX
import { useLoaderData } from 'react-router-dom';

function User() {
  const userData = useLoaderData(); 
  return (
    <div>
      <h1>{userData.name}</h1>
      <p>Email: {userData.email}</p>
    </div>
  );
}

```
The difference between using `useEffect` and `loader` is that `loader` runs before the component is rendered.
